# Project instructions

## Knowledge base

Before changing the project, read `README.md`, `docs/gdd.md`, and
`docs/technical-documentation.md`. Treat the documented P5 course requirements
as authoritative. Report conflicts instead of silently changing scope.

## Engineering rules

- Keep all player-facing game text and the presentation in English. Keep the
  project documentation, reports, README, and test forms in Croatian.
- Preserve the locked vertical-slice scope: one level, three weapons, three
  standard enemy types, and one two-phase boss.
- Run `./scripts/validate.sh` after code changes.
- Record new assets in both asset licence manifests before using them.
- Document material AI assistance in `docs/ai-usage.md`.
- Never place credentials, personal data, production logs, or confidential
  material in prompts, source files, or the repository.
